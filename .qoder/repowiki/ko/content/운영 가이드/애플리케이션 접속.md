## 소개
이 문서는 prj-devops 환경에서 배포된 주요 애플리케이션에 접근하는 방법을 안내합니다. plate-web, plate-api, plate-llm 등의 애플리케이션에 대해 Ingress를 통해 제공되는 도메인 정보를 기반으로 HTTPS로 접속하는 절차를 설명합니다. 또한 ArgoCD 대시보드와 Harbor 레지스트리 포털에 접근하는 방법을 포함하며, 초기 비밀번호 추출 및 보안 설정 변경에 대한 절차도 안내합니다.

## 도메인 기반 HTTPS 접속

prj-devops 환경에서는 Ingress를 통해 각 애플리케이션에 도메인 기반으로 접근할 수 있습니다. 각 애플리케이션은 별도의 도메인 또는 서브도메인을 통해 HTTPS로 서비스됩니다.

- **plate-web (프로덕션)**: `https://cocdev.co.kr` 또는 `https://www.cocdev.co.kr`
- **plate-api (프로덕션)**: `https://cocdev.co.kr/api` 또는 `https://www.cocdev.co.kr/api`
- **plate-llm (스테이징)**: `https://llm.cocdev.co.kr`

이러한 도메인 설정은 `ingress-prod.yaml` 및 각 애플리케이션의 ArgoCD Application 정의 파일에 명시되어 있으며, cert-manager를 통해 Let's Encrypt 인증서가 자동 발급되어 안전한 HTTPS 연결을 보장합니다.

## ArgoCD 대시보드 접근

ArgoCD 대시보드는 GitOps 기반의 애플리케이션 배포 상태를 모니터링하고 관리하는 데 사용됩니다. 관리자 계정으로 로그인하기 위해 초기 비밀번호를 추출해야 합니다.

### 초기 비밀번호 추출
ArgoCD 설치 후 생성되는 초기 관리자 비밀번호는 Kubernetes Secret에 저장되어 있으며, 다음 명령어를 통해 확인할 수 있습니다:

```bash
kubectl -n argocd get secret argocd-initial-admin-secret -o jsonpath="{.data.password}" | base64 -d
```

이 비밀번호는 `argocd/templates/NOTES.txt` 파일에 안내된 절차에 따라 추출되며, 첫 로그인 후 반드시 변경해야 합니다.

## Harbor 레지스트리 포털 접속

Harbor는 프라이빗 컨테이너 이미지 레지스트리로, 애플리케이션 이미지를 저장하고 관리하는 데 사용됩니다. Harbor 포털에 접속하려면 다음 절차를 따르세요.

### 접속 URL 확인
Harbor 포털의 접속 주소는 Helm 차트의 `values.yaml`에 설정된 `externalURL` 값에 따라 결정됩니다. 설치 후 다음과 같은 URL로 접근할 수 있습니다:

```
https://harbor.cocdev.co.kr
```

이 정보는 `harbor/templates/NOTES.txt` 파일에 안내되어 있으며, 설치 완료 후 몇 분 정도 기다린 후 접속해야 합니다.

## 보안 권장 사항

보안을 강화하기 위해 다음과 같은 조치를 권장합니다:

1. **ArgoCD 초기 비밀번호 변경**: 첫 로그인 후 반드시 초기 비밀번호를 변경하고, 강력한 비밀번호 정책을 적용하세요.
2. **Harbor 관리자 계정 보호**: 기본 관리자 계정(admin)의 비밀번호를 변경하고, 다단계 인증(MFA)을 활성화하세요.
3. **정기적인 비밀번호 갱신**: 모든 관리자 계정에 대해 정기적인 비밀번호 변경 정책을 수립하고 적용하세요.
4. **RBAC 설정**: 사용자 역할에 따라 접근 권한을 세분화하여 불필요한 접근을 방지하세요.

이러한 보안 조치는 시스템의 무단 접근을 방지하고, 운영 환경의 안정성을 유지하는 데 필수적입니다.
